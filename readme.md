# 🎄 3D 互动圣诞之光 (高级定制版)

这是一个基于 **React 18**、**Three.js** 和 **AI 手势识别**的 3D 圣诞树互动项目。
它专为非技术用户设计，支持**零代码修改图片**，并能通过简单的文件编辑来自定义祝福寄语。

---

## 🛠️ 第一步：准备环境 (仅需 3 分钟)

在开始之前，请确保您的电脑安装了 **Node.js**（这是运行网页项目的引擎）。

1. **下载安装**：前往 [Node.js 官网](https://nodejs.org/)，下载并安装 **LTS (长期支持版)**。
2. **验证安装**：打开终端（Windows 搜索 `cmd`，Mac 搜索 `终端`），输入以下命令并回车：
   ```bash
   node -v
   ```
   *看到类似 `v18.x.x` 或 `v20.x.x` 的数字，说明成功了！*

---

## 📦 第二步：启动项目

1. **解压项目**：将下载的压缩包解压缩到一个文件夹中。
2. **进入文件夹**：在终端中使用 `cd` 命令进入该文件夹，或者在该文件夹路径栏输入 `cmd` 打开。
3. **安装依赖**（仅需第一次）：
   ```bash
   npm install
   ```
4. **本地预览**：
   ```bash
   npm start
   ```
   *稍等片刻，浏览器会自动打开 `http://localhost:3000`，您就能看到 3D 圣诞树了！*

---

## 🎨 第三步：自定义您的内容 (核心部分)

这是您准备给同事惊喜的关键步骤。**所有的修改都在 `public` 文件夹中进行**，修改后保存文件并刷新网页即可生效。

### 1. 换成您自己的照片 📸
1. 打开 `public/photo/` 文件夹。
2. **删除**里面的测试文件。
3. **放入照片**：将您和同事的照片、合影、公司 Logo 等放入该文件夹。
4. **重命名**：文件必须以数字开头，例如：
   - `1.jpg`
   - `2.png`
   - `3.jpeg`
   - `4.webp`
   - *提示：建议放 6-20 张，系统会自动把它们挂在树上。*

### 2. 修改祝福语和标题 ✍️
1. 用记事本（或编辑器）打开 `public/config.json` 文件。
2. 修改对应内容：
   ```json
   {
     "treeTitle": "给XX部门的一份礼物",   // 网页上方的大标题
     "signatureText": "祝大家圣诞快乐，明年不加班！", // 点击照片后显示的寄语
     "autoProbeCount": 20                // 系统自动寻找照片的最大数量
   }
   ```
3. **保存文件**。

## 🎮 如何交互？

您可以告诉您的同事有以下两种玩法：

*   **玩法 A：AI 手势控制 (需要摄像头)** ✋
    - 屏幕右下角会显示摄像头画面。
    - **握拳**：圣诞树会紧缩，代表“收纳记忆”。
    - **张开手掌**：圣诞树会爆炸式散开，代表“释放惊喜”。
    - **移动手掌**：树体会跟随您的手掌旋转，并产生光影律动。
*   **玩法 B：鼠标/触摸控制 (通用)** 🖱️
    - **拖拽**：左右旋转圣诞树。
    - **滚轮/双指缩放**：放大看照片细节。
    - **点击照片**：树上的“拍立得”照片会被拉近，并显示您在 `config.json` 里写的祝福。

## 🚀 第四步：发送给同事 (部署)

当您在本地修改满意后，需要生成一个可以在互联网访问的链接：

1. **打包项目**：
   在终端输入：
   ```bash
   npm run build
   ```
   *运行完后，您的项目文件夹里会出现一个名为 `dist` (或 `build`) 的文件夹。*

2. **部署建议**：
   - **Vercel (推荐)**：最简单的免费托管。只需将项目上传到 GitHub，然后在 Vercel 关联即可，它会给您一个 `https://...` 的链接。
   - **GitHub Pages**：将代码推送到仓库，在 Settings 中开启 Pages 服务。
   - **局域网分享**：如果您和同事在同一个 Wi-Fi 下，您运行 `npm start` 后，将您的 IP 地址（如 `http://192.168.1.5:3000`）发给他，他也能直接打开。

## ❓ 常见问题排查

- **Q：照片没显示出来？**
  - A：请检查照片名是否正确（如 `1.jpg` 而不是 `照片1.jpg`），且文件确实放在 `public/photo/` 下。
- **Q：摄像头没反应？**
  - A：请在浏览器地址栏左侧点击“锁”图标，确保已允许该网页使用摄像头。
- **Q：画面太卡？**
  - A：由于 3D 粒子较多，如果是旧款笔记本，请尝试关闭不必要的浏览器标签页。
